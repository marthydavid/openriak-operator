# Agent Instructions for OpenRiak Operator Development

This document provides guidance for agents working on the OpenRiak Operator project.

## Project Overview

The OpenRiak Operator is a Kubernetes operator for managing Riak clusters. It's built with kubebuilder and provides CRDs for:
- **RiakCluster**: Main cluster resource
- **RiakBucket**: Bucket provisioning
- **RiakUser**: User and ACL management

## Key Architecture Points

### Controllers
- Located in `internal/controller/`
- Each CRD has a corresponding reconciler
- Controllers use the manager module for Riak operations

### Riak Management
- Located in `internal/riak/`
- `executor.go`: Executes kubectl exec to run riak-admin commands
- `manager.go`: High-level operations (join nodes, create buckets, etc.)

### Shell-Based Management
**Important**: Riak does NOT have an HTTP API. All management operations go through:
```bash
kubectl exec -n <namespace> <pod> -c riak -- riak-admin <command>
```

This is executed via the Executor in `internal/riak/executor.go`.

## Common Tasks

### Add New Riak Operation
1. Add method to `Executor` in `internal/riak/executor.go` (low-level)
2. Add method to `Manager` in `internal/riak/manager.go` (high-level)
3. Use in controller's Reconcile method

### Fix Controller Issues
1. Check controller logs: `kubectl logs -n riak-system -l app.kubernetes.io/name=openriak-operator`
2. Add status conditions to track state
3. Use finalizers for cleanup operations

### Test Changes
```bash
make generate      # Generate CRDs and RBAC
make manifests     # Generate K8s manifests
make docker-build  # Build operator image
make deploy IMG=<your-image>
```

## CRD Design Patterns

All CRDs follow this pattern:
```go
type <Resource>Spec struct {
    // Desired state fields
}

type <Resource>Status struct {
    Phase <ResourcePhase>
    Conditions []metav1.Condition
    Error string
    LastUpdateTime *metav1.Time
}

type <ResourcePhase> string

const (
    PhaseCreating <ResourcePhase> = "Creating"
    PhaseReady    <ResourcePhase> = "Ready"
    PhaseFailed   <ResourcePhase> = "Failed"
)
```

## Unit Testing

Unit tests live in `internal/controller/` (envtest) and `internal/riak/` (plain Go tests).

Coverage target ≥ 85%. Run:
```bash
go test ./internal/... -coverprofile cover.out -timeout 180s
go tool cover -func cover.out
```

### Executor injection for controller tests

`Executor *riak.Executor` is an injectable field on `RiakBucketReconciler` and `RiakUserReconciler`.
When nil a real kubectl-based executor is used. In tests inject a no-op runner:

```go
noopRunner := func(_ context.Context, _ string, _ ...string) (string, error) { return "", nil }
r := &RiakBucketReconciler{
    Client:   k8sClient,
    Scheme:   k8sClient.Scheme(),
    Executor: riak.NewExecutorWithRunner(logr.Discard(), noopRunner),
}
```

### CRD validation — dead code to avoid

`Grant.Resource` is `+kubebuilder:validation:Enum=bucket;any`. Empty string is rejected by the API
server before it reaches any controller, so do NOT add guards for empty resource in controller code.

## Container Image Registry

All images live at **`ghcr.io/marthydavid/`** — do not use DockerHub (`openriak/`) paths.

| Image | Full reference |
|-------|---------------|
| Operator | `ghcr.io/marthydavid/openriak-operator:<tag>` |
| Riak KV 3.2 | `ghcr.io/marthydavid/riak:3.2.6` |

Build and publish images locally:
```bash
make docker-build-riak                          # builds ghcr.io/marthydavid/riak:3.2.6
make docker-push-riak
make docker-build IMG=ghcr.io/marthydavid/openriak-operator:dev
make docker-push  IMG=ghcr.io/marthydavid/openriak-operator:dev
```

CI builds are handled by two GitHub Actions workflows:
- `.github/workflows/build-operator.yml` — operator image, multi-arch (amd64+arm64), triggers on push to `main` and `v*` tags
- `.github/workflows/build-riak.yml` — Riak image (amd64 only, RPM is x86_64), triggers when `images/riak/**` changes

The Riak Dockerfile is at `images/riak/Dockerfile` (UBI9 base, RPM sourced from
`https://files.tiot.jp/riak/kv/3.2/`).

## Important Notes

1. **Pod Anti-Affinity**: Riak nodes must run on different hosts for fault tolerance. This is configured in the StatefulSet spec.

2. **Headless Service**: The cluster communication uses a headless service (`<cluster>-headless`). This is critical for inter-node communication.

3. **Status Updates**: Always update cluster status to track progress. Controllers poll for status changes.

4. **Finalizers**: Used for graceful deletion. Remove in deletion handlers.

5. **Configuration**: Riak config is passed via environment variables to the container. The container startup script (in the Riak image) sets up riak.conf from these variables.

## Debugging Tips

- Always check if cluster is Ready before attempting operations
- Watch pods: `kubectl get pods -l cluster=<name> -w`
- Check riak-admin commands: `kubectl exec -it <pod> -c riak -- riak-admin status`
- Review controller events: `kubectl describe riakcluster <name>`

## Security Review

### Agent Pipeline (`/security-review`)

The skill uses a 3-step agent pattern:

1. **Explore agent** — reads `executor.go`, `manager.go`, controllers, `entrypoint.sh`, and integration scripts to identify candidate vulnerabilities
2. **Parallel Explore agents** (one per candidate) — apply the FALSE POSITIVE FILTERING rules independently
3. **Main agent** — drops findings with confidence < 8/10

### Known False Positive Patterns

**Command injection via kubectl exec** — always false positive in this codebase.
`executor.go` uses `exec.CommandContext(ctx, name, args...)` with argument-array semantics — no shell is invoked. Shell metacharacters in bucket names, usernames, or passwords are passed as literal strings.

**`RIAK_CONFIG_*` environment variable injection** — false positive.
These env vars come exclusively from `RiakCluster.spec.riakConfig`, which requires Kubernetes RBAC write access. Attackers who can write RiakCluster resources already have cluster-level access; the injection vector is moot.

**Config value logging** — false positive.
`SetConfig` logs Riak node configuration key-value pairs only. User passwords flow through a separate `CreateUser()` path and are never passed to `SetConfig`.

### Known Real Vulnerability

**Hardcoded default password** (`internal/controller/riakuser_controller.go:139`):
`passwordSecret` is optional in the CRD schema. When absent, the controller sets the Riak user password to `"changeme"` via the full executor chain. Fix: make `passwordSecret` required (`+kubebuilder:validation:Required`) or auto-generate a random secret.
